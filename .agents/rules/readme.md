# README Style Guidelines

When adding/editing a feature in the `README.md` for this project, always adhere to the following rules:

1. **Keep it Minimal**: Avoid long, verbose paragraphs or extensive prose. Get straight to the point.
2. **List Features Concisely**: Always explain the features of the script(s), but do so using a concise bulleted list (e.g., under a "Features" heading).
3. **Show Core Commands**: Clearly present the core execution commands (like scanner vs. interactive cleanup) in syntax-highlighted code blocks.
4. **No Fluff**: Ensure the overall footprint of the file remains as minimal as possible while remaining highly informative.
